## Frontmatter 规范

所有页面必须包含以下 frontmatter 字段：

```yaml
---
title:          # 页面标题（必填）
created:        # 创建日期 YYYY-MM-DD（必填）
updated:        # 更新日期 YYYY-MM-DD（必填）
type:           # 页面类型：entity | concept | comparison | query | raw | review | meta（必填）
tags:           # 标签列表（必填，至少一个）
sources:        # 来源引用列表（选填，格式：[来源ID]
---
```

## 页面阈值

### 新建页面条件
- 一个独立知识点（如一个实验范式、一个统计概念）
- 一对易混淆概念的对比
- 一道高频真题或典型 query
- 一篇 raw 原始资料（article/paper/transcript）

### 并入现有页面条件
- 新内容与已有页面主题完全相同 → **不覆盖**，在 sources 中追加引用
- 同一概念的两个版本描述 → 在 updated 字段更新，保留旧版本在 log.md 中记录
- 相关小概念 → 并入父级概念页面（如"内隐记忆"并入"记忆"实体）

---

## Update Policy（更新策略）

1. **不直接覆盖已有内容**：所有更新通过追加 update log 实现
2. **冲突信息处理**：
   - 新旧信息并存，标注 `updated: YYYY-MM-DD`
   - 在页面底部添加 `## 更新记录` 小节，记录变更原因
3. **版本追溯**：log.md 记录每次重大变更
4. **raw/ 目录永不修改**：原始资料只追加，不编辑
